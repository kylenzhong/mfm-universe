## Top 3 buildable opportunities

### 1. A packaged "skills + memory" harness that ships with a curated local-model stack
- **Category / stage:** Developer infrastructure / early-and-heating. Today's #1 and #2 AI repos are both harnesses (ECC 224,320; hermes-agent 206,782), not models.
- **Velocity / cross-validation:** the entire GitHub AI top ten is agent plumbing, and it's cross-validated by the HF side — the harnesses want cheap swappable models underneath, and downloads on local GGUFs are surging (`Unlimited-OCR` +201K/day, `Qwythos` past 1.1M). The two boards are describing one stack from opposite ends. Guest fit: **Dharmesh Shah (0.90, agent tooling / B2B SaaS)** and **Greg Isenberg (0.80)**.
- **Hypothesis:** every solo builder now hand-rolls the same scaffolding — skill definitions, a memory store, a security boundary, model routing — because the popular harnesses (ECC, hermes-agent) target Claude Code / Cursor power users, not the "give me a working agent on my own model in one command" crowd. The wrapper is commoditized; the *opinionated default bundle* isn't.
- **MVP (≤1 week):** a one-command harness that ships with a preselected local model per task lane (OCR/reasoning/coding), a file-backed memory store, a skills folder convention, and a security sandbox — `npx my-agent init` → working local agent. Steal ECC's "skills + instincts + memory" taxonomy as the interface.
- **Why now:** the harness layer just became the most-starred thing in open-source AI *today*; the abstraction is validated but the packaging for non-power-users is wide open.
- **Confidence: 4/5.**

### 2. On-device document OCR pipeline built on `Unlimited-OCR`
- **Category / stage:** Applied AI / nascent, but the velocity is the strongest single number on the board.
- **Velocity / cross-validation:** `baidu/Unlimited-OCR` added **~201K downloads in one day** (429K → 630,246) and gained likes (1,415 → 1,524) — the fastest-moving individual model on HF today, and it's an OCR/vision model, i.e. a *product primitive*, not a chat toy. Guest fit: **Brett Adcock (0.82, on-device vision)**, **Demis Hassabis (0.76)**, **Pieter Levels (0.88, local/on-device)**.
- **Hypothesis:** small teams drowning in scanned PDFs / receipts / forms still pay per-page cloud OCR (Textract, Google Vision) and leak the documents to a third party. A local, zero-marginal-cost OCR service that runs the current best open model solves both cost and privacy — and the "current best" just got measurably better and free.
- **MVP (≤1 week):** a dockerized `POST /ocr` endpoint wrapping `Unlimited-OCR` with layout-preserving output and a batch folder-watch mode. One demo: drop 100 scanned invoices in a folder, get structured JSON out, no cloud call.
- **Why now:** the model doing the +201K-downloads-a-day is an OCR model — the market is voting with pulls, and running it locally is now genuinely viable on consumer hardware.
- **Confidence: 3/5.**

### 3. A lane-specialist local-model router (carryover — the download acceleration strengthens it)
- **Category / stage:** Developer tooling / nascent. Held from yesterday; the "why now" got sharper.
- **Velocity / cross-validation:** the board is still one-winner-per-lane and now the winners are *pulling away on downloads* — OCR (`Unlimited-OCR` 630K), roleplay (`Qwythos` 1.1M), reasoning (`Ornith-1.0-35B` 233,701), coding (`gemma-4-12B-agentic-fable5-composer` 288,741), agents (`Qwen-AgentWorld-35B-A3B` 34,371). ollama's own repo description still names `Kimi-K2.6, GLM-5.1, MiniMax, DeepSeek` — the default runtime is again a cycle behind the board. Guest fit: **Pieter Levels (0.88, local LLM)**.
- **Hypothesis:** a builder hard-codes one base model and never revisits, leaving a 2×-better specialist unpulled every week; the cost of not tracking the board is now measurable because a different model demonstrably wins each lane.
- **MVP (≤1 week):** a daily job that pulls the top ~50 trending GGUFs, classifies them into ~8 lanes, ranks each by download-velocity + likes, and emits a one-line `ollama pull` per lane plus a thin router that dispatches to the current winner.
- **Why now:** the release cadence has outrun manual re-evaluation and the download-velocity gap between lane-winners and the field widened again today.
- **Confidence: 4/5.**
