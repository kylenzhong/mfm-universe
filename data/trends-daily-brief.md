## Top 3 buildable opportunities

The board got re-pointed today — not by fresh quant, but because the news rewrote the premises. That's a better reason to move than another day of stale velocity. Honest re-rank below.

### 1. "Own your AI memory" — a portable, auditable memory-export layer

- **What / stage** — a tool that pulls a user's accumulated memory/profile out of ChatGPT (and Claude, Gemini, Character.AI) into a portable, human-readable JSON the user controls, with a one-screen "here's what the AI silently decided to remember about you" audit view. Consumer AI / data-portability. The category went from theoretical to live *this week*.
- **Velocity / cross-validation** — Dreaming V3 (June 4) makes profiling automatic and free-tier; the 96%-unilateral-memory finding is the privacy headline; EU AI Act transparency obligations take effect **August 2026**; Anthropic's S-1 (June 1) explicitly leans on ad-free + behavioral-permanence as its differentiation narrative. Four independent vectors, all this week, all pointing at "the AI knows things about you that you never told it to keep."
- **Hypothesis** — as memory goes automatic and invisible, the felt problem flips from "make the AI remember me" to "show me — and let me move — what it remembers." No incumbent will build a clean *export* button; it's anti-retention for them. That's the durable indie wedge.
- **MVP** — a static page + thin serverless fn: user pastes a ChatGPT memory export (Settings → Personalization → Manage Memory is already user-exportable), we parse + categorize + flag the unilateral/stale entries, emit a portable `memory.json`, and offer "rehydrate this into a Claude project." Ship the parser on Opus 4.8. ≤1 week. Measure 72h signups off one r/OpenAI + one r/privacy thread.
- **Why now** — the feature that creates the problem shipped 48 hours ago and the EU transparency clock starts in ~8 weeks. First mover defines "AI memory portability" before the incumbents add a deliberately-ugly export flow.
- **Confidence: 4/5** *(this is the standing off-ramp thesis, re-based onto the stronger, fresher premise Dreaming V3 just handed it).*

### 2. Companion-app compliance SDK — still buildable, now on a 25-day Colorado clock with explicit federal tail-risk

- **What / stage** — drop-in middleware for companion/character chatbots: age-gating, "I am not a licensed professional" disclosure, crisis-language detection + hand-off, audit log. Builder infra / regtech.
- **Velocity / cross-validation** — Colorado's AI Act (first comprehensive US state AI law) takes effect **June 30 — 25 days**; the state-AG companion-liability dragnet (PA's fake-medical-claims suit in active discovery, plus the FL/KY actions the June 5 brief flagged) is widening; Character.AI has already settled the Garcia + four other wrongful-death suits and barred under-18 back-and-forth chat. BUT — today's material change — the **Great American AI Act** (269-page discussion draft, June 4, Obernolte/Trahan) proposes a **three-year preemption of state AI laws.** If it advances, the 50-state-patchwork rationale that made a single SDK valuable partially collapses.
- **Hypothesis** — near-term (next ~60 days) the Colorado deadline + live AG enforcement force every companion app to adopt *something*; a clean drop-in wins on time-to-compliance. Longer-term the federal-preemption fight is the risk to underwrite, not ignore.
- **MVP** — npm package: `ageGate()`, `disclosure()`, `crisisDetect()` over an open classifier + a hosted audit-log endpoint. Free <10K MAU, $0.001/check above. Classifier on Opus 4.8.
- **Why now** — 25 days to the first real enforcement deadline. The preemption bill is a *discussion draft, not an introduced bill* — a long, litigable road — so the near-term forcing function is intact even though the long-term moat just got a question mark.
- **Confidence: 3/5** *(held — the Colorado clock keeps it alive; the preemption draft caps the upside).*

### 3. Memory-portability's B2B twin — a "bring-your-context" SDK for agent builders

- **What / stage** — the same memory-export primitive from opp 1, packaged for builders: a small SDK that lets any agentic app import a user's portable `memory.json` so users land with their context pre-loaded instead of cold-starting. Builder infra.
- **Velocity / cross-validation** — MCP is now donated to a foundation and OpenAI adopted Agent Skills, so *interop at the standard layer is real* — but nobody owns the *user-context* portability layer specifically. Dreaming V3 (OpenAI), Claude projects, and Gemini personalization are each walled gardens of profile data. Cheap frontier models (Qwen 3.7 Max ~½ Opus input cost, MiniMax M2.7 at $0.30/$1.20) mean switching costs are now *only* context, not capability — which makes portable context the actual lock-in battleground.
- **Hypothesis** — when models are commoditized and interchangeable on price, the thing that keeps a user is their accumulated context. A neutral portability standard for that context is the high-value, un-owned layer. Pairs with opp 1 (consumer wedge feeds the B2B standard).
- **MVP** — publish the `memory.json` schema from opp 1 as an open spec + a 50-line import helper; get one indie agent app to adopt "Sign in with your memory." Measure: one external adopter in 2 weeks.
- **Why now** — the standards layer (MCP/Skills) just consolidated; the *context* layer is the next obvious one and is currently empty.
- **Confidence: 3/5** *(new; speculative but rides the same evidence as opp 1 and the model-commoditization data).*
