## Top 3 buildable opportunities

### 1. A lane-specialist model router — "the best local model for *this* task, this week," auto-refreshed
- **Category / stage:** Developer tooling / nascent — and today is the sharpest evidence yet.
- **Velocity / cross-validation:** the board fragmented hard. Five different fine-tunes own five different lanes and *all* gained over the weekend: `baidu/Unlimited-OCR` (1,415 / 429k) for OCR, `Qwythos-9B-Claude-Mythos` (992 / 970k) for roleplay, `Ornith-1.0` (new today) for reasoning, `gemma-4-12B-coder-fable5-composer` (2,510 / 575k) for code, `Qwen-AgentWorld-35B` for tool-use. No two lanes share a winner and the winners rotate weekly. Guest fit: **Pieter Levels (0.88, local LLM / on-device)** — cost-aware, offline, own-your-stack, exactly this user.
- **Hypothesis:** a builder picking a local model hard-codes one base on vibes and never revisits, leaving a 2×-better specialist on the table every week. The cost of *not* tracking the board is now measurable because the board demonstrably has a better answer per lane.
- **MVP (≤1 week):** script that pulls the top ~50 trending GGUFs, classifies them into ~8 task lanes, ranks each by download-velocity + likes, and outputs a daily-refreshing one-line `ollama pull` per lane. Optionally a thin router that dispatches a prompt to the current lane-winner.
- **Why now:** the single-best-model era ended this morning — the board visibly fragmented into specialists, and they're all moving at once. The recurring question "which local model for my one task, today" went from quarterly to weekly.
- **Confidence: 4/5.**

### 2. "Ship the eval with the model" — auto-generate a lane benchmark alongside any fine-tune
- **Category / stage:** Developer infrastructure / brand-new signal (first seen clearly today).
- **Velocity / cross-validation:** `Qwen` published `Qwen-AgentWorld-35B-A3B` **and** `Qwen/AgentWorldBench` (55 likes, fresh) as a pair — the model and the rubric it's graded on, shipped together. This is the natural endgame of a fragmenting board: when every lane has its own winner, the differentiator is a *credible, lane-specific benchmark*, and the lab that ships one defines the scoreboard. Nobody packages this for the long tail of fine-tuners.
- **Hypothesis:** thousands of fine-tuners (the Fable-5 / gemma-composer crowd) ship a model with a one-line "trained on X traces" claim and no eval, so buyers can't trust it and downloads stay vibes-driven. A tool that, given a fine-tune + its task lane, auto-generates a held-out benchmark and a verifiable scorecard turns "trust me" into "here's the number."
- **MVP (≤1 week):** CLI that takes a model + a lane tag, synthesizes/pulls a held-out eval set for that lane, runs the model vs. the lane incumbent, and emits a publishable `BENCH.md` scorecard (the AgentWorldBench format as the template). One demo: benchmark `gemma-4-12B-coder-fable5` on coding vs. its base.
- **Why now:** Qwen just made "model + benchmark as a bundle" the prestige move today; the pattern is fresh and unowned for everyone below the frontier labs.
- **Confidence: 3/5.**

### 3. A day-zero "should I swap?" harness for new open-weight drops
- **Category / stage:** Developer infrastructure / nascent. Carryover — and the "why now" got stronger, not weaker.
- **Velocity / cross-validation:** `ollama`'s own GitHub description (175,168 stars) **still** advertises `Kimi-K2.6 / GLM-5.1` — now two-plus cycles stale after GLM-5.2 and today's Ornith/AgentWorld drops. The default runtime cannot keep pace; the agent-harness repos above it (ECC 223,658, hermes-agent 206,041) grind upward but don't solve evaluation. "A new frontier/specialist weight just dropped — is it actually better for *my* lane?" is now a multiple-times-a-week event with no tooling.
- **Hypothesis:** builders re-benchmark approximately never because it's a manual afternoon each time, so they chase shiny drops blindly or ossify on a stale base. A harness that auto-runs *your* task suite on any new top-of-board weight and emits a swap/hold verdict turns the afternoon into a notification.
- **MVP (≤1 week):** CLI that watches the Hub like/download board, detects a new lane-class drop (Ornith-1.0 today), pulls it via ollama, runs a user-supplied eval set against the incumbent, prints `SWAP` / `HOLD` + per-lane pass-rate delta.
- **Why now:** the release cadence (GLM-5.2, then Ornith + AgentWorld within days) has officially outrun manual re-evaluation, and the default runtime is provably two versions behind.
- **Confidence: 4/5.**
